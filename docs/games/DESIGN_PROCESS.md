# Game Design Process

**Purpose**: Standard workflow for designing new games  
**Author**: Game Design Team  
**Last Updated**: 2025-11-09  
**Status**: Active  

---

## Design Workflow

### 1. Concept Phase
- Identify cognitive skill to train
- Research existing games and mechanics
- Define core gameplay loop
- Assess feasibility and scope

### 2. Design Phase
- Create game design document (use template)
- Define rules and scoring
- Design UI mockups
- Plan async multiplayer implementation

### 3. Review Phase
- Design review with team
- Cognitive science validation
- Technical feasibility check
- Approval from stakeholders

### 4. Development Phase
- Implement game logic
- Build UI components
- Integrate with platform
- Write tests

### 5. Testing Phase
- Internal playtesting
- User testing (target personas)
- Balance and difficulty tuning
- Bug fixing

### 6. Launch Phase
- Soft launch to subset of users
- Monitor metrics and feedback
- Iterate and improve
- Full rollout

## Design Checklist

Before starting development:
- [ ] Game design document complete
- [ ] Cognitive benefits validated
- [ ] Async multiplayer designed
- [ ] UI mockups approved
- [ ] Technical approach agreed
- [ ] Success metrics defined
